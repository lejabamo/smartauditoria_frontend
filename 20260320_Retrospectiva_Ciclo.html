📋 1. Evidencias de Progreso

Backend (Flask API)

  • Gestión de Activos: Implementado en activos.py. Soporte para CRUD y clasificación CIA.
  • Modelado de Riesgos: Engine de cálculo Probabilidad x Impacto integrado en riesgos.py.
  • Motor RAG v1: Servicio híbrido en iso_suggestions_service.py para controles ISO.
  • Autenticación: Sistema JWT funcional con roles roles_auth.

Frontend (React)

  • Dashboard: Visualización inicial de métricas clave.
  • Wizard: Identificación guiada integrada con el backend.
  • Matriz 5x5: Visualización dinámica de criticidad.

Gestión de Proyecto

  • Jira: Backlog sincronizado y organizado en épicas.
  • Agentes: Jira, Dev y RAG operativos en el workspace.

🧠 2. Retrospectiva del Equipo

  • Fortalezas: Arquitectura de microservicios robusta y entrega ágil de sugerencias v1.
  • Desafíos: Necesidad de mayor precisión semántica; migración a Vector DB obligatoria.
  • Ajuste Espiral: Replanificación exitosa de las épicas EP1 y EP2 para asegurar calidad.
